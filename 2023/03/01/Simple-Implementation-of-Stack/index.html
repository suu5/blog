Simple Implementation of Stack

Posted on 2023-03-01 Edited on 2023-03-20

栈的简单实现

栈

设计思路

  • 线性结构的栈将使用两种存储方式:顺序存储和链式存储
  • 设计基类对外提供栈的基础操作函数，设计两个派生类分别完成顺序存储和链式存储的具体实现
  • 调用出栈函数后，应注意顺序栈存放的类型，若为类类型，需要用户另外对对象进行处理，顺序栈出栈不会自动析构对象

基类设计

  • 基类Stack作为抽象类，提供如下的函数

初始化栈

  • 此处提供的初始化函数分别对应两种派生类，作为纯虚函数，需要在派生类中分别重写，对具体实现的栈而言，两个函数只能有一个起作用，调用另外一个初始化函数将返回错误
  • Status Stack::InitStack(unsigned _size, unsigned _increment);
  • Status Stack::InitStack();

入栈

  • Status Stack::Push(T _element);

出栈

  • Status Stack::Pop(T &_element);

获取栈顶元素

  • Status Stack::GetTop(T &_element);

判空

  • bool Stack::StackEmpty();

清空栈

  • Status Status::ClearStack();

销毁栈

  • Status Stack::DestroyStack();

派生类设计

顺序栈

  • SequentialStack
  • 使用数组作为存储结构，内存空间分布于堆区
  • 使用无符号整型变量表示顺序栈的容量，增长空间，栈顶(数组下标形式，非指针类型)
  • 栈空的标志：栈顶 = 0，栈满的标志：栈顶 = 数组尾后下标
  • 栈需要先通过调用”InitStack(unsigned, unsigned)”函数初始化其容量和增长空间后才能使用
  • 当顺序栈满时，再次入栈将通过私有成员变量”increment”拓展数组空间，若该变量设置为0，表示栈无扩容能力
  • 当调用清空栈和销毁栈函数时，程序将使用”delete []”关键字释放堆区上的资源

链栈

  • ChainStack
  • 使用链表作为存储结构，内存空间分布与堆区
  • 定义”struct StackNode“作为链表节点
  • 链表头节点不储存数据，仅作为栈底标识
  • 栈需要先通过调用”InitStack()”函数初始化，分配栈底节点
  • 当调用清空栈和销毁栈函数时，程序将循环使用”delete”关键字释放节点资源
